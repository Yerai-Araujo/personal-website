Este proyecto es la evolución del proyecto Asset tracker a una arquitectura de microservicios. El objetivo es demostrar cómo una aplicación monolítica puede ser transformada en un conjunto de servicios independientes que se comunican entre sí, mejorando la escalabilidad, mantenibilidad y flexibilidad del sistema. Cada microservicio se encarga de una funcionalidad específica, como gestión de usuarios, seguimiento de activos o cálculo de valor del portfolio.

Funcionalidades principales

  • Todas las funcionalidades del proyecto monolítico, pero implementadas en un entorno de microservicios.

Arquitectura de Microservicios

El sistema está compuesto por varios microservicios, se detallan las funcionalidades de cada uno:

  • User
    • Responsabilidades: registro, autenticación, JWT, gestión de usuarios
    • Tiene su propia base de datos
  • Portfolio
    • Responsabilidades: seguimiento de precios y gestión de activos disponibles
    • Tiene su propia base de datos
  • Market
    • Responsabilidades: Obtener precios externos, Cache, Retry, CircuitBreaker, Fallback
    • No necesita su propia base de datos, solo cache
